TickerQ is the modern background job scheduler for .NET. It uses Roslyn source generators to register job functions at compile time — no reflection, no magic strings, fully AOT and trimming compatible. Add jobs with a single attribute, persist them to your existing database, and monitor everything in real time.

## Get started in three steps

<Steps>
  <Step title="Install TickerQ">
    Add the core package and a persistence provider to your .NET project.

    ```bash
    dotnet add package TickerQ
    dotnet add package TickerQ.EntityFrameworkCore
    ```
  </Step>
  <Step title="Register and define a job">
    Wire up services and decorate any method with `[TickerFunction]`.

    ```csharp Program.cs
    builder.Services.AddTickerQ(options =>
    {
        options.AddOperationalStore(ef =>
            ef.UseTickerQDbContext<TickerQDbContext>(db =>
                db.UseSqlite("Data Source=tickerq.db")));
    });

    var app = builder.Build();
    app.UseTickerQ();
    ```

    ```csharp Jobs.cs
    public class MyJobs
    {
        [TickerFunction("SendWelcomeEmail")]
        public async Task SendWelcomeEmail(
            TickerFunctionContext context,
            CancellationToken cancellationToken)
        {
            Console.WriteLine($"Job {context.Id} running at {context.ScheduledFor:O}");
        }
    }
    ```
  </Step>
  <Step title="Schedule a job">
    Use `ITimeTickerManager` to schedule a one-off job, or define a recurring cron expression directly on the attribute.

    ```csharp
    public class SchedulingService(ITimeTickerManager<TimeTickerEntity> manager)
    {
        public async Task ScheduleWelcomeEmail()
        {
            await manager.AddAsync(new TimeTickerEntity
            {
                Function = "SendWelcomeEmail",
                ExecutionTime = DateTime.UtcNow.AddSeconds(10)
            });
        }
    }
    ```
  </Step>
</Steps>

## Packages

TickerQ is split into focused packages so you only take what you need.

| Package | Purpose |
|---|---|
| `TickerQ` | Core scheduler engine and background services |
| `TickerQ.Utilities` | Shared types, entities, interfaces, and attributes |
| `TickerQ.EntityFrameworkCore` | EF Core persistence (PostgreSQL, SQL Server, SQLite, MySQL) |
| `TickerQ.Caching.StackExchangeRedis` | Redis persistence and distributed coordination |
| `TickerQ.Dashboard` | Real-time SignalR monitoring dashboard |
| `TickerQ.Instrumentation.OpenTelemetry` | OpenTelemetry activity tracing |
| `TickerQ.SourceGenerator` | Compile-time job function registration (included automatically) |

<Note>
  Always keep all TickerQ packages on the same version. They are versioned and released together.
</Note>
