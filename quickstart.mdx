---
title: Quick start
description: Get TickerQ running in your .NET application in under 5 minutes.
---

This guide walks through installing TickerQ, wiring up EF Core persistence, defining a job, and scheduling it. The example follows the pattern used in the official sample applications.

<Steps>

<Step title="Install packages">

Add the core scheduler and the EF Core persistence provider to your project.

```bash
dotnet add package TickerQ
dotnet add package TickerQ.EntityFrameworkCore
```

`TickerQ.SourceGenerator` is included automatically as a transitive dependency of `TickerQ`. You do not need to install it separately.

<Tip>
  To add the real-time monitoring dashboard, also install `TickerQ.Dashboard` and call `options.AddDashboard()` inside `AddTickerQ()`. See [Dashboard setup](/dashboard/setup) for details.
</Tip>

</Step>

<Step title="Register services">

Call `AddTickerQ()` in your service registration and configure the operational store.

<CodeGroup>

```csharp ASP.NET Core (Program.cs)
using Microsoft.EntityFrameworkCore;
using TickerQ.DependencyInjection;
using TickerQ.EntityFrameworkCore.DbContextFactory;
using TickerQ.EntityFrameworkCore.DependencyInjection;

var builder = WebApplication.CreateBuilder(args);

builder.Services.AddTickerQ(options =>
{
    options.AddOperationalStore(efOptions =>
    {
        efOptions.UseTickerQDbContext<TickerQDbContext>(dbOptions =>
        {
            dbOptions.UseSqlite(
                "Data Source=tickerq.db",
                b => b.MigrationsAssembly("MyApp"));
        });
    });
});

var app = builder.Build();

// Apply the TickerQ schema
using (var scope = app.Services.CreateScope())
{
    var db = scope.ServiceProvider.GetRequiredService<TickerQDbContext>();
    db.Database.Migrate();
}
```

```csharp Generic host / Console (Program.cs)
using Microsoft.EntityFrameworkCore;
using Microsoft.Extensions.DependencyInjection;
using Microsoft.Extensions.Hosting;
using TickerQ.DependencyInjection;
using TickerQ.EntityFrameworkCore.DbContextFactory;
using TickerQ.EntityFrameworkCore.DependencyInjection;
using TickerQ.Utilities;

var host = Host.CreateDefaultBuilder(args)
    .ConfigureServices((context, services) =>
    {
        services.AddTickerQ(options =>
        {
            options.AddOperationalStore(efOptions =>
            {
                efOptions.UseTickerQDbContext<TickerQDbContext>(dbOptions =>
                {
                    dbOptions.UseSqlite(
                        "Data Source=tickerq.db",
                        b => b.MigrationsAssembly("MyConsoleApp"));
                });
            });
        });
    })
    .Build();

// Apply the TickerQ schema
using (var scope = host.Services.CreateScope())
{
    var db = scope.ServiceProvider.GetRequiredService<TickerQDbContext>();
    db.Database.Migrate();
}

// Build function metadata for non-web hosts
TickerFunctionProvider.Build();
```

</CodeGroup>

`AddTickerQ()` registers the scheduler background services, managers, and the in-memory fallback persistence. Passing an `AddOperationalStore` call replaces the in-memory fallback with EF Core.

</Step>

<Step title="Create a job">

Decorate any method with `[TickerFunction]` and give it a unique name. The source generator picks it up at compile time.

```csharp Jobs.cs
using TickerQ.Utilities.Base;

public class MyJobs
{
    [TickerFunction("HelloWorld")]
    public Task HelloWorldAsync(
        TickerFunctionContext context,
        CancellationToken cancellationToken)
    {
        Console.WriteLine(
            $"Hello from TickerQ! Id={context.Id}, ScheduledFor={context.ScheduledFor:O}");
        return Task.CompletedTask;
    }
}
```

`TickerFunctionContext` carries runtime information about the execution: the job ID, its scheduled time, the retry count, and the ticker type (time or cron). You can also receive a typed request payload by using `TickerFunctionContext<TRequest>`.

For recurring jobs, pass a cron expression directly on the attribute:

```csharp Jobs.cs
[TickerFunction("DailyReport", cronExpression: "0 9 * * *")]
public async Task SendDailyReport(
    TickerFunctionContext context,
    CancellationToken cancellationToken)
{
    // Runs every day at 09:00
}
```

</Step>

<Step title="Activate the scheduler">

Call `UseTickerQ()` to signal the initializer and start the scheduler.

<CodeGroup>

```csharp ASP.NET Core (Program.cs)
app.UseTickerQ();
app.Run();
```

```csharp Generic host / Console (Program.cs)
host.UseTickerQ();
await host.RunAsync();
```

</CodeGroup>

`UseTickerQ()` must be called after `Build()`. All I/O-bound startup work (function discovery, database seeding, provider startup) is deferred to `TickerQInitializerHostedService`, which runs when the host starts. Design-time tools that build the host without starting it are never affected.

</Step>

<Step title="Schedule a job">

Inject `ITimeTickerManager<TimeTickerEntity>` wherever you need to schedule a one-off job.

<CodeGroup>

```csharp ASP.NET Core — minimal API endpoint
using TickerQ.Utilities.Entities;
using TickerQ.Utilities.Interfaces.Managers;

app.MapPost("/schedule", async (ITimeTickerManager<TimeTickerEntity> manager) =>
{
    var result = await manager.AddAsync(new TimeTickerEntity
    {
        Function = "HelloWorld",
        ExecutionTime = DateTime.UtcNow.AddSeconds(5)
    });

    return Results.Ok(new { result.Result.Id, ScheduledFor = result.Result.ExecutionTime });
});
```

```csharp Generic host — hosted service
using TickerQ.Utilities.Entities;
using TickerQ.Utilities.Interfaces.Managers;

public class SampleScheduler : IHostedService
{
    private readonly ITimeTickerManager<TimeTickerEntity> _manager;

    public SampleScheduler(ITimeTickerManager<TimeTickerEntity> manager)
    {
        _manager = manager;
    }

    public async Task StartAsync(CancellationToken cancellationToken)
    {
        var result = await _manager.AddAsync(new TimeTickerEntity
        {
            Function = "HelloWorld",
            ExecutionTime = DateTime.UtcNow.AddSeconds(5)
        }, cancellationToken);

        if (!result.IsSucceeded)
        {
            Console.WriteLine($"Failed to schedule job. Exception: {result.Exception}");
            return;
        }

        Console.WriteLine(
            $"Scheduled job Id={result.Result.Id}, ScheduledFor={result.Result.ExecutionTime:O}");
    }

    public Task StopAsync(CancellationToken cancellationToken) => Task.CompletedTask;
}
```

</CodeGroup>

`Function` must exactly match the name you passed to `[TickerFunction]`. The scheduler polls for due jobs and executes them using the registered background services.

</Step>

</Steps>
